# ADR-0018: Agent Tool for Sub-Agent Spawning

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Agent/Swarm
**Context:** `src/tools/AgentTool/`, feature flag `FORK_SUBAGENT`

## Context

Complex tasks benefit from decomposition into sub-tasks that can be handled by independent agents.

## Decision

Implement an AgentTool that spawns sub-agents with their own context and tool access, controlled by the `FORK_SUBAGENT` feature flag.

## Consequences

### Positive
- Task decomposition
- Parallel execution capability
- Isolated sub-agent contexts

### Negative
- Resource overhead per sub-agent
- Context synchronization complexity
- Cost multiplication

# ADR-0019: Swarm Team Architecture

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Agent/Swarm
**Context:** `src/tools/TeamCreateTool/`, `src/tools/TeamDeleteTool/`, `src/tools/SendMessageTool/`, `src/utils/swarm/`

## Context

Multi-agent collaboration requires team management and inter-agent communication.

## Decision

Implement swarm tools:
- **TeamCreateTool** - Create agent teams
- **TeamDeleteTool** - Remove teams
- **SendMessageTool** - Inter-agent communication
- **Swarm utilities** - Team coordination helpers

## Consequences

### Positive
- Coordinated multi-agent workflows
- Explicit team management
- Structured inter-agent communication

### Negative
- Complex orchestration logic
- Debugging multi-agent interactions
- Resource management challenges
