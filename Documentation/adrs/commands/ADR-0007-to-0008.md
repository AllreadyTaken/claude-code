# ADR-0007: Three-Tier Command System

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Commands
**Context:** `src/commands.ts`, `src/commands/`

## Context

User-facing commands need different execution models - some expand to prompts, some execute locally, some render UI.

## Decision

Implement three command types:
- **`prompt`** - Expands to a prompt sent to the model (used by skills)
- **`local`** - Executes locally, returns text (e.g., /clear, /help)
- **`local-jsx`** - Renders React UI in the terminal (e.g., /config, /model)

## Consequences

### Positive
- Clear separation of command behaviors
- JSX commands enable rich interactive UI
- Prompt commands integrate with AI naturally

### Negative
- Three execution paths to maintain
- Type safety across command types is complex

# ADR-0008: Command Registry with Memoization

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Commands
**Context:** `src/commands.ts`, `src/commands/`

## Context

70+ commands need to be loaded, filtered, and accessed efficiently.

## Decision

Use a central command registry with `lodash-es/memoize` for caching expensive command loading operations.

## Consequences

### Positive
- Fast command lookup
- Lazy command loading
- Reduced startup cost

### Negative
- Memoization cache invalidation complexity
- Memory overhead for cached commands
