# ADR-0001: Async Generator Query Loop

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Core AI/Query
**Context:** `src/query.ts`, `src/QueryEngine.ts`

## Context

The core interaction with the Anthropic API requires streaming responses while maintaining state across multiple turns (user message, assistant response, tool execution, tool result, next assistant message).

## Decision

Use an async generator function (`query()`) that yields messages (stream events, assistant, user, progress, attachments, system) and returns a terminal state.

## Consequences

### Positive
- Streaming responses with immediate UI updates
- Stateful turn management within a single function
- Natural pause/resume points via generator semantics
- Clean separation of streaming vs final state

### Negative
- Complex control flow with yield points
- Error handling must account for generator state
- Debugging generator state is non-trivial

# ADR-0002: Context Compaction Strategies

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Core AI/Query
**Context:** `src/services/compact/`

## Context

LLM context windows have finite capacity. Long conversations with extensive tool results can exceed the token limit.

## Decision

Implement multiple compaction strategies:
- **Auto-compact:** Triggered at context threshold
- **Reactive compact:** Triggered on prompt-too-long errors
- **Micro-compact:** Cached editing-based granular compaction
- **Snip:** Feature-gated history snipping
- **Context collapse:** Feature-gated granular collapsing

## Consequences

### Positive
- Flexible strategies for different scenarios
- Feature flags allow gradual rollout
- Reactive handling prevents hard failures

### Negative
- Multiple strategies increase complexity
- Compaction can lose important context
- Feature flag proliferation

# ADR-0003: Zero Runtime Dependencies

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Core AI/Query
**Context:** `package.json`

## Context

Distribution and installation of the CLI tool should be as simple and conflict-free as possible.

## Decision

Bundle all code into a single `cli.js` file with zero runtime dependencies in `package.json`. Only optional platform-specific native modules (`@img/sharp-*`) are listed.

## Consequences

### Positive
- No dependency conflicts with user's project
- Fast, reliable installation
- Single file distribution
- Simplified version management

### Negative
- Large binary size
- Cannot patch individual dependencies
- All dependencies hidden (harder to audit)
- Rebuild required for any dependency update
