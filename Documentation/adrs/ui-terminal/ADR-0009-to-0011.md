# ADR-0009: React/Ink for Terminal UI

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** UI/Terminal
**Context:** `src/ink/`, `src/components/`, `src/screens/`

## Context

Terminal UI needs component composition, state management, and efficient rendering for an interactive CLI experience.

## Decision

Use Ink (React-based terminal UI framework) with 144 components, 85 hooks, and 3 main screens (REPL, Doctor, ResumeConversation).

## Consequences

### Positive
- Component composition for complex UI
- React ecosystem (hooks, context, state)
- Familiar paradigm for web developers
- Declarative UI updates

### Negative
- React overhead in terminal environment
- Custom reconciler needed for terminal
- Learning curve for terminal-specific patterns

# ADR-0010: Custom Ink Terminal Framework

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** UI/Terminal
**Context:** `src/ink/` (48 files)

## Context

Standard Ink doesn't provide all needed terminal capabilities (ANSI parsing, focus management, custom rendering).

## Decision

Build a custom Ink-based framework with:
- Custom reconciler and renderer
- Focus management
- ANSI/termio parsing (CSI, OSC, ESC, DEC sequences)
- Text wrapping, measurement, layout

## Consequences

### Positive
- Full control over terminal rendering
- Custom ANSI sequence handling
- Optimized for specific use case

### Negative
- Significant maintenance burden
- Custom framework knowledge required
- 48 files of framework code to maintain

# ADR-0011: Screen-Based Navigation

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** UI/Terminal
**Context:** `src/screens/`

## Context

The application has distinct modes that require different UI layouts and interactions.

## Decision

Implement three main screens:
- **REPL** - Primary interactive mode
- **Doctor** - Diagnostics and troubleshooting
- **ResumeConversation** - Session recovery

## Consequences

### Positive
- Clear screen boundaries
- Focused UI per mode
- Easy to add new screens

### Negative
- Screen switching complexity
- Shared state management across screens
