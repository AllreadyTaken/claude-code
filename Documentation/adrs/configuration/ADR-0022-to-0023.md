# ADR-0022: Multi-Source Settings Resolution

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Configuration
**Context:** `src/utils/settings/settings.ts`, `src/constants/`

## Context

Settings come from multiple sources (global config, project config, CLI flags, environment) and need consistent resolution.

## Decision

Implement multi-source settings resolution with priority ordering:
1. CLI flags (highest)
2. Environment variables
3. Project settings
4. Global settings
5. Defaults (lowest)

## Consequences

### Positive
- Flexible configuration
- Override capability at each level
- Clear precedence rules

### Negative
- Complex resolution logic
- Debugging which source provided a value
- Settings conflict handling

# ADR-0023: Configuration Migrations

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Configuration
**Context:** `src/migrations/` (11 migration scripts)

## Context

Configuration schema evolves over time, requiring migration of existing user configurations.

## Decision

Implement versioned configuration migrations with 11 migration scripts that run during initialization.

## Consequences

### Positive
- Backward compatibility
- Automatic schema evolution
- User config preservation

### Negative
- Migration code accumulates
- Testing all migration paths
- Migration failure handling
