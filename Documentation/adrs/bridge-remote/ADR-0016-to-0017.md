# ADR-0016: WebSocket-Based Remote Bridge

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Bridge/Remote
**Context:** `src/bridge/` (31 files), `src/remote/`

## Context

Users need to control Claude Code from web/mobile interfaces, requiring bidirectional communication with permission callbacks.

## Decision

Implement a bridge system with:
- WebSocket-based transport
- Poll-based fallback
- Bidirectional permission callbacks
- Session management with JWT auth

## Consequences

### Positive
- Real-time bidirectional communication
- Authenticated sessions via JWT
- Fallback for restricted environments
- Remote permission handling

### Negative
- WebSocket connection management complexity
- Network reliability concerns
- Security considerations for remote access
- 31 files of bridge code

# ADR-0017: Direct Connect Session Server

**Status:** Accepted
**Date:** 2026-04-01
**Domain:** Bridge/Remote
**Context:** `src/server/`, feature flag `DIRECT_CONNECT`

## Context

Some deployments require direct session management without intermediary services.

## Decision

Implement a direct connect session server behind the `DIRECT_CONNECT` feature flag.

## Consequences

### Positive
- Direct session control
- Reduced latency
- No external service dependency

### Negative
- Additional server code to maintain
- Feature flag complexity
- Security surface area increase
